# Thoughts Persistence

Proactively persist plans, research, and reviews to the thoughts directory.

## Directory

```
thoughts/
├── <profile>/           # repo-specific notes (most common)
│   ├── plans/           # implementation plans, design docs
│   ├── research/        # investigations, evaluations, comparisons
│   └── reviews/         # code reviews, architectural reviews
├── global/<profile>/    # cross-repo notes (architecture decisions, tooling preferences)
└── searchable/          # hard links for search tools (auto-generated, read-only)
```

`<profile>` is your HumanLayer profile name. Resolve it once with `ls thoughts/` — it's the single non-`searchable`, non-`global` entry — and use that path throughout.

Filename: `YYYY-MM-DD-{slug}.md` (lowercase, hyphenated).

Frontmatter:
```yaml
---
source: {repo basename or "global"}
date: YYYY-MM-DD
type: {plan|research|review}
---
```

## Canonical Paths

All reads and writes use `thoughts/<profile>/` or `thoughts/global/<profile>/`.

`searchable/` is read-only — a search index regenerated by `humanlayer thoughts sync`. When a search result returns `thoughts/searchable/<profile>/plans/2026-03-01-foo.md`, the canonical path is `thoughts/<profile>/plans/2026-03-01-foo.md`.

## Detection

- `thoughts/<profile>/` exists → initialized, persist automatically.
- `thoughts/` missing → tell the user the symlink farm isn't set up and to run:
  ```bash
  humanlayer thoughts init --directory {repo}
  ```
  Do not assume where `thoughts/` links to. `humanlayer thoughts init --help` documents profile options. `humanlayer thoughts status` shows sync state.
